C++虚函数

发表于2023-06-13|C++

虚函数

  1. 纯虚函数基类可以被实例化吗

    纯虚函数基类不能被实例化。纯虚函数是在基类中声明为虚函数，并且被赋予 = 0 的函数。当一个类中包含纯虚函数时，这个类就成为抽象类（或称为接口类）。抽象类的主要作用是作为一个基类，为派生类提供一个共同的接口，但自身不能被实例化。

    由于纯虚函数没有具体的实现，因此包含纯虚函数的类（即抽象类）不能被实例化。尝试实例化一个抽象类会导致编译错误。这是因为抽象类的主要目的是定义一种行为或接口，而不是作为一个具体的对象存在。具体的实现细节应该由派生类来完成。

  2. 派生类析构时是如何调用基类的虚构函数的？

    首先会执行派生类自身的析构函数体中的代码，然后自动调用其基类的析构函数。

    编译器会确保在派生类析构函数的末尾，按照基类在派生类继承列表中的逆序（从右向左，从下到上）自动调用基类的析构函数。

  3. 如果一个派生类继承了两个基类，两个基类中均有虚函数，派生类有几个虚函数表？

    如果一个派生类继承了两个基类，并且这两个基类都有虚函数，那么派生类通常只有一个虚函数表（vtable），会包含所有从基类继承来的虚函数，以及派生类自身定义的任何虚函数。但每个基类指针或引用会维护自己的vptr来确保正确地调用虚函数。

    这是因为虚函数表是类的一个属性，而不是每个基类的一个独立属性。当派生类继承基类的虚函数时，这些虚函数会被整合到派生类的虚函数表中。

    派生类的虚函数表这个虚函数表是共享的，无论是通过哪个基类指针或引用访问派生类对象，都会使用同一个虚函数表来解析虚函数调用。

  4. 构造函数可以包含虚函数吗

    在C++中，构造函数本身不能是虚函数。虚函数主要用于动态多态性，允许在基类中声明一个函数，并在派生类中提供不同的实现。通过基类指针或引用来调用派生类对象上的虚函数时，会执行派生类中的实现。

    然而，构造函数在对象创建和初始化时调用，此时对象的类型（基类还是派生类）是已知的，因此不需要动态绑定。此外，在构造函数被调用时，对象的虚函数表（vtable）可能还没有完全设置，因此调用虚函数可能是未定义的。

    虽然构造函数不能直接是虚函数，但构造函数内部可以调用虚函数。然而，这样的调用不会表现出多态行为，因为此时调用的将是构造对象类型的虚函数版本，而不是可能通过基类指针或引用调用的派生类版本。

    通常，在构造函数中调用虚函数是不推荐的，因为这可能会导致意料之外的行为和逻辑错误。如果需要在基类构造函数中执行与派生类相关的某些操作，更好的做法是使用初始化列表来调用派生类的特定初始化函数，或者将相关逻辑放在其他非虚的成员函数中。

    总之，构造函数不能是虚函数，且在构造函数中调用虚函数通常是不安全的，也是不推荐的。
