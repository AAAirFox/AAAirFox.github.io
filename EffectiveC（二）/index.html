05 了解 C++ 默默编写并调用哪些函数

  1. 什么时候空类不再是一个空类？

  2. 四个函数的特性：

    都是public且inline的（条款30）。

    • 默认构造函数：给编译器放置调用基类/成员变量函数的代码。

    • 默认析构函数是非虚的（条款07），除非基类的析构函数是虚函数。

    • 拷贝构造函数/拷贝赋值操作符：单纯将每一个非静态成员变量拷贝。

      参数为(const string& s, const T& value)时，如果传入("hello", 3)，第一个参数类型为string，会调用string库内的copy函数构造；第二个参数类型为int，是内置类型，会拷贝传入值的每一个bit来构造。

      =操作符函数不生成的情况：成员变量为引用/const时，不符合C++特性；或基类中=操作符的声明为private，但派生类=需要处理基类成分（条款12），无法被派生类调用。编译器不生成此函数。

08 别让异常逃离析构函数

  • 析构函数绝对不要吐出异常。如果一个被析构函数调用的函数可能抛出异常，析构函数应该捕捉任何异常，然后吞下它们(不传播)或结束程序。
  • 如果客户需要对某个操作函数运行期间抛出的异常做出反应，那么class应该提供一个普通函数(而非在析构函数中)执行该操作。

C++**析构函数最好不要吐出异常**：如果一个vector中有数个元素销毁时异常，那么会抛出n个异常。在两个异常同时存在的情况下，程序若不是结束执行就是导致不明确行为，最终可能会调用std::terminate终止执行。

09 绝不在构造和析构过程中调用 virtual 函数

在构造派生类时，会先调用基类的构造函数，如果在其中调用了虚函数，派生类函数尚未被初始化，对象的类型是正在构造的基类的类型（虚函数表 + 当专属成分尚未被初始化时，视为不存在），无法动态绑定到派生类虚函数。

同理析构函数，当派生类析构执行后，当前对象被视为基类类型。

虚函数可能隐藏在多重调用内，如构造()-> init()-> virtual log()，此时编译器和连接器通常不会报错，极有可能调用错误版本的log()。

11 在 operaotr= 中处理“自我赋值”

潜在的自我赋值：a[i] = a[j]l;

如果两个对象来自同一个继承体系，不需要声明为相同类型，就可能造成“别名”因为基类的引用或指针可以指向派生类对象：Base& rb = Drived *pd-》rb和*pd可能是同一对象。

如果遵循条例13和14，运用对象管理资源+copy时举措正确，=可能是自我赋值安全的。但如果自行管理，可能会在停止使用资源之前意外释放。

解决方案：

  1. 增加证同测试(==判断地址)，如果是同一个，就不处理。

  2. 语句顺序趋势：不处理自我赋值，重点放在异常安全上。

    如果new时抛出异常，pb保持现状。

  3. copy and swap

确定任何函数如果操作一个以上的对象，而其中多个对象是同一个对象时，其行为仍然正确。

12 复制对象时勿忘其每一个成分

  • copy操作应确保复制所有变量及所有基类成分（调用基类copy）。
  • 不要尝试用某个copy函数（赋值与构造两种）实现另一个copy函数，应该将共同部分放入第三个函数中，由两个copy共同调用。

大多数编译器不会为少复制了成员变量而报错（条款53）：既然用户自己写了copy，那代码不完全他们也不管。

如果派生类需要赋值其中基类部分，如果基类部分变量为private，那么需要调用基类的对应copy。
