Win10安装PostgreSQL9.6：Unable to write inside TEMP environment variable path

发表于2020-03-16

小张今天上数据库的课，装postgreSQL9.6.1，信心十足地下载exe打开，结果安装程序都妹打开直接报错。
![太难受了](./Win10安装PostgreSQL9.6：Unable to write inside TEMP environment variable path/20200316130855431.png)
清过空间、看过权限没问题后小张开始善用百度。
解答文章挺多，有些用注册表修改：【终结方案】 Windows下安装PostgreSQL 9.0.4：unable to write inside temp environment variable path
但小张失败，找不到.vbs

最后参考了简书的这一篇文章：
unable to write inside temp environment variable path
和CSDN的这一篇文章：
解决：输入错误: 没有文件扩展“.vbs”的脚本引擎
终于解决问题。快乐去恰饭。

  1. 我的电脑->右键属性->高级系统设置->高级->环境变量，看temp文件夹路径。
    ![小张的页面](./Win10安装PostgreSQL9.6：Unable to write inside TEMP environment variable path/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L3dlaXhpbl80NDc3NTg4Nw==,size_16,color_FFFFFF,t_70.png)

  2. 清空一下。temp存放系统临时文件，可以删除。

  3. 再次运行SQL安装程序，仍然显示TEMP问题。

  4. 打开temp文件夹，找到最新生成的log文件，打开后可以看到错误具体信息：
    ![开头可以看到生成的具体时间等信息](./Win10安装PostgreSQL9.6：Unable to write inside TEMP environment variable path/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L3dlaXhpbl80NDc3NTg4Nw==,size_16,color_FFFFFF,t_70-1712315069870-5.png)

    可见找不到.vbs的脚本引擎。
    上面简书文章在注册表中找到HKEY_CLASSES_ROOT\CLSID文件夹下后缀为{XXX-00AA004A55E8}的文件夹，将InprocServer32的路径修改为C:\Windows\system32\vbscript.dll
    但小张注册表中这个路径本来就没有问题，只好另外查找解决办法。

  5. 采用CSDN文章中添加.vbs连接的方法。
