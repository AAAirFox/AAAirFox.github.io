10. 和为K的子数组

考虑使用滑动窗口（错误：[0, 0, 0]）

[i, j]区间的和 = [0, j]的和 - [0, i - 1]的和

  • 暴力解

    枚举左右边界（包括left=right），求左右边界之间的sum，判断。

  • 暴力解优化（时间长空间少）

    当左边界不动时，右边界像右移动一个，则在 j 的sum基础上+nums[j+1]。判断。

    当有边界不动时，左边每次从有边界加一个。判断。

  • 前缀和：构建前缀和数组，以快速计算区间和；下标有偏移。（性能基本无提升）

    第0处的前缀值为0，第i处的前缀值存储在preSum[i]值为为preSum[i-1] + nums[i-1].

    [0, j]的和：j+1的前缀值 [0, i - 1]的和：i的前缀值

  • 前缀和+哈希表：只关心次数，不关心具体解

    把出现过的子数组保存到哈希表中，就可以O(1)的时间查找。

    在i处计算有多少个，将i视为右边界，在i处（包括i）的前缀和sum = 左边界前缀和（不包括左边界）+target ==> 左边界前缀和 = sum - target。

    求在i处有多少个，即为查找有多少左边界前缀和的值为sum-target。

11. 滑动窗口的最大值

  • 窗口固定暴力循环，注意值可能为负数，max的初始值应为负数的最小值。

    若向右滑动的新元素≥max，直接更新，否则重新计算。（超出时间限制）

  • 滑动窗口中，最左边的数会逐渐移出窗口，且当前窗口内最大值存在时，其左边所有窗口内的其他数值永远不会成为下一个窗口的最大值，所以可以立即放弃。所以队列中最左边值永远是最大的。

    当窗口有了新的值，如果它比现在队列中的第一个值大，则新值左边所有数字可以移出。

    如果没有第一个值大，则将新值左边所有小于等于它的数去除，将新值放入队列之中。

    由于有左右移除的操作需求，可以选择双端队列；由于从队首到队尾单调递减，具有单调性，选择单调队列。

    用队列存储nums每个值对应的Index。

  • 及时去掉无用数据，保证双端队列有序

    时间：O(n)

    空间：O(min(size, 不重复的数值个数))

22. 相交链表

是链表指针相等不是值相等。

  • 由于相交之后不会再分开，所以只可能在较短一条链上相交，且两条链尾部对齐。从短链head开始判断，如果不相等则后移1位，如果相等则当前节点开始相交。

  • 【双指针】让2个指针走一样的距离，消除步行差，如果相交，一定到达相交点。

    A=不相交A前缀链 + 相交后链C

    B=不相交B前缀链 + 相交后链C

    A：A链 - C - B链 - C

    B：B链 - C - A链 - C

    当两个节点相等时，要么走到了C链开头，要么走到了结尾null。所以直接返回当前节点即为结果。

    如果A长度是n个B——同样：

    A：A链 - C - B链 - C

    B：B链 - C - A链 - C

  • 将链表B构成环，如果相交，则在A中进行快慢指针时会发现有环；如果不相交，A中没有环。在A中有环时，把快慢指针其中一个指向headA，再相遇即为成环的地方。

96. 只出现一次的数字

  • 异或运算

    既满足时间复杂度又满足空间复杂度，就要提到位运算中的异或运算 XOR，主要因为异或运算有以下几个特点：
    一个数和 0 做 XOR 运算等于本身：a⊕0 = a
    一个数和其本身做 XOR 运算等于 0：a⊕a = 0
    XOR 运算满足交换律和结合律：a⊕b⊕a = (a⊕a)⊕b = 0⊕b = b

97. 多数元素

  • 摩尔投票法

    推论一： 若记 众数 的票数为 +1 ，非众数的票数为 -1，则一定有所有数字的票数和 >0 。

    推论二：若数组的前 a 个数字的票数和 =0 ，则 数组剩余 (n-a) 个数字的 票数和一定仍 >0 ，即后(n-a) 个数字的 众数仍为 x 。

    根据以上推论，记数组首个元素为n1，众数为x，遍历并统计票数。当发生票数和=0 时，剩余数组的众数一定不变 ，这是由于：

    当 n1 =x ：抵消的所有数字中，有一半是众数x。
    当 n1≠x： 抵消的所有数字中，众数 x 的数量最少为 0 个，最多为一半。
    利用此特性，每轮假设发生票数和 =0 都可以 缩小剩余数组区间 。当遍历完成时，最后一轮假设的数字即为众数。
