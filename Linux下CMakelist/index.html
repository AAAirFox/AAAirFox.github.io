Linux下CMakeList

发表于2023-05-09|C++

参考链接：

使用

具体函数

enable_language()

enable_language() 函数在 CMake 中用于启用对指定编程语言的支持，其可以设置的参数包括：

  • CXX：启用对C++的支持。
  • C：启用对C的支持。
  • Fortran：启用对Fortran的支持。
  • Asm：启用对汇编语言的支持。

示例

腾讯libco

  1. 设置cmake版本号

    cmake_minimum_required(VERSION 2.8)

  2. 定义项目名称

    project(libco)

  3. 对于mac系统，设置是否在macOS上使用RPATH.

    此变量用于初始化所有目标上的MACOSX_RPATH属性。

    set(CMAKE_MACOSX_RPATH 0)

  4. 设置库的版本号

    set(LIBCO_VERSION 0.5)

  5. 设置C编译器编译的标志，调试信息、优化级别、警告等。

    set(CMAKE_C_FLAGS ${CMAKE_C_FLAGS} -g -fno-strict-aliasing -O2 -Wall -export-dynamic -Wall -pipe -D_GNU_SOURCE -D_REENTRANT -fPIC -Wno-deprecated -m64)

    • -g：生成调试信息
    • -fno-strict-aliasing：关闭严格别名规则，允许使用不严格的指针别名。
    • -O2：启用二级优化，以提高生成的代码的性能。
    • -Wall：启用所有警告选项，报告可能的问题。
    • -export-dynamic：在共享库中导出动态符号，以便其他模块可以在运行时查找和使用它们。
    • -pipe：启用管道优化，将编译过程分为多个阶段，提高编译速度。
    • -D_GNU_SOURCE：定义宏_GNU_SOURCE，告诉编译器使用GNU C标准库中的特定功能和扩展。
    • -D_REENTRANT：定义宏_REENTRANT，告诉编译器支持可重入函数和线程安全性。
    • -fPIC：生成位置无关代码（Position Independent Code），使生成的代码可以在内存中的任何位置运行。
    • -Wno-deprecated：禁用关于已弃用函数和特性的警告。
    • -m64：为64位体系结构生成代码。

  6. 启用C和汇编语言支持

    enable_language(C ASM)

  7. 添加源文件列表

    set(SOURCE_FILES

    ​ co_epoll.cpp

    ​ co_hook_sys_call.cpp

    ​ co_routine.cpp

    ​ coctx.cpp

    ​ coctx_swap.S)

  8. 添加静态库、动态库

    add_library(colib_static STATIC ${SOURCE_FILES}) add_library(colib_shared SHARED ${SOURCE_FILES})

    设置目标属性，设置输出库的名字为colib，即文件名为libcolib。

    set_target_properties(colib_static PROPERTIES OUTPUT_NAME colib)
    set_target_properties(colib_shared PROPERTIES OUTPUT_NAME colib)

    设置目标属性，设置在清理构建时，直接删除目标的输出文件，而不是通过依赖关系间接删除。【CLEAN_DIRECT_OUTPUT不是CMake的标准属性】

    set_target_properties(colib_static PROPERTIES CLEAN_DIRECT_OUTPUT 1)

    set_target_properties(colib_shared PROPERTIES CLEAN_DIRECT_OUTPUT 1)

    设置动态库的版本号和符号链接。

    “soname”是一个符号链接（symlink）的名称，它指向实际的共享库文件。这个symlink允许在不更改已编译的二进制文件的情况下，用新的共享库文件替换旧的共享库文件，只要新的库文件保持与旧的库文件相同的ABI版本。

    set_target_properties(colib_shared PROPERTIES VERSION ${LIBCO_VERSION} SOVERSION ${LIBCO_VERSION})

  9. 定义一个宏，用于添加示例目标

  1. 使用宏添加多个示例目标，为括号内生成对应的可执行文件，并将其链接到指定库上。

    对于每一个调用，CMake会创建一个example_[EXAMPLE_TARGET]可执行文件，将对于源文件example_[EXAMPLE_TARGET].cpp编译为可执行文件，将可执行文件链接到colib_static/pthread/dl库。

    add_example_target(closure)

    add_example_target(cond)

    add_example_target(copystack)

    add_example_target(echocli)

    add_example_target(echosvr)

    add_example_target(poll)

    add_example_target(setenv)

    add_example_target(specific)

    add_example_target(thread)
