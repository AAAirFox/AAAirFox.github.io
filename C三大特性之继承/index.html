C++三大特性之继承

继承

让某种类型对象获得另⼀个类型对象的属性和⽅法。

目的：它可以使⽤现有类的所有功能，并在⽆需重新编写原来的类的情况下对这些功能进⾏扩展

方式：

  • 实现继承：指使⽤基类的属性和⽅法⽽⽆需额外编码的能⼒
  • 接口继承：指仅使⽤属性和⽅法的名称、但是⼦类必须提供实现的能⼒
  • 可视继承：指⼦窗体（类）使⽤基窗体（类）的外观和实现代码的能⼒
访问限定符 基类->派生类
public
公有继承		 基类的public成员 -> 子类的public成员
基类的protected成员 -> 子类的protected成员
基类的private成员只能通过基类的接口访问
protected
保护继承		 基类的public成员 -> 子类的protected成员
基类的protected成员 -> 子类的protected成员
基类的private成员只能通过基类的接口访问
private
私有继承		 基类的public成员 -> 子类的private成员
基类的protected成员 -> 子类的private成员
基类的private成员只能通过基类的接口访问
注意： protected成员只能在类内部或子类内部访问；
使用访问限定符智能收缩权限：
公有继承时，基类的公用成员和保护成员在派生类中保持原有的访问属性，其私有成员仍为基类私有，即在派生类中不能访问，在类外也不能访问。私有成员体现了数据的封装性，如果基类的私有成员可以被派生类所访问，即破坏了基类的封装性，这就会失去C++的一个重要特性。

派生类总是可以转换为基类的类型引用，基类转换为派生类需要在确认安全的情况下，使用强制类型转换static_cast/dynamic_cast。

  1. 为什么派生类不能访问基类的私有成员？

    派生类和基类是不同的类域，在不同类中访问相当于在类外访问私有成员。在派生类的成员函数中访问了父类的私有成员，相当于在类外访问私有成员，是错误的。

  2. 派生类无论怎样都无法访问基类的私有成员，怎么获取私有成员的状态？

    在基类定义非私有的get函数，然后在派生类直接getX()调用即可。

  3. 构造一个派生类的步骤：

    • 从基类接受成员：全部，不含构造析构。

    • 调整接受的成员：调整访问限制；重新声明函数覆盖。

    • 在声明派生类时增加成员。

  4. 多重继承时的结构（顺序相关）：

友元

友元关系不能被继承。

友元关系是单向的，不具有交换性。

友元关系不具备传递性。

派生类的friend函数可以访问派生类本身的一切变量，包括从父类继承下来的protected域中的变量。但是对父类来说，他并不是friend的。

友元函数

提高程序的效率，通过减少通过接口访问的限制，为不同类之间的成员函数以及类的成员函数与一般函数之间提供数据共享的接口。这使得友元函数能够直接访问类的私有和保护成员，即使它不是类的成员函数。

  • 使用友元而不使用public的原因：

    1. 效率考虑：有时，通过public函数访问private成员可能会引入不必要的函数调用开销。
    2. 访问多个私有成员：如果一个函数需要访问多个private成员，并且这些成员之间存在一定的关联或依赖关系，那么将该函数声明为友元可能会更简洁和方便。通过友元函数，你可以直接访问这些成员，而无需通过多个public函数进行中转。
    3. 操作符重载：为了重载某些操作符（如<<用于输出流），你可能需要将操作符函数声明为类的友元。这样，该函数就可以直接访问类的私有成员，以便正确地实现操作符的行为。
    4. 两个类之间的紧密耦合：当两个类之间存在紧密的依赖关系，并且一个类需要直接访问另一个类的私有成员时，友元函数可能是一个合适的解决方案。然而，这种情况应该尽量避免，因为它可能导致代码难以理解和维护。

  • 友元函数不是类的成员函数，因此没有隐式的this指针。

    友元函数可以在类的外部定义，而且其定义与普通的非成员函数定义没有区别。

  • 如果需要访问非静态成员，可以通过对象引用或指针来访问。

    void printSecret(MyClass& obj);

  • 友元关系破坏了封装性和隐藏性，因此应谨慎使用友元函数。

  • 一个类可以有多个友元函数，每个友元函数都需要单独声明。

  • 如果两个类在同一作用域使用了同名、同初始化列表的友元函数，会导致错误。

友元类

允许一个类（友元类）访问另一个类（宿主类）的私有（private）和保护（protected）成员。这种关系突破了封装性的限制，使得友元类可以直接访问宿主类的内部状态。

静态成员函数

在基类和派生类中，其静态成员函数与静态成员是共用一段空间的，即静态成员和静态成员函数是可以继承的。

父类的static变量和函数在派生类中依然可用，但是受访问性控制（比如，父类的private域中的就不可访问）。而且对static变量来说，派生类和父类中的static变量是共用空间的，这点在利用static变量进行引用计数的时候要特别注意。

其他成员函数问题

在派生类中定义一个同名函数，编译器是将同名函数隐藏了，不管参数表是否相同。即不会构成函数重载，直接为函数覆盖。

函数覆盖/隐藏

函数覆盖（等于重写，多态的一种）

函数重载的条件是在同一个作用域中才会构成函数重载，而派生类和基类是两个类域，所以一定不会构成函数重载。

覆盖是指派生类函数覆盖基类函数：

（1）不同的范围（分别位于派生类与基类）；
（2）函数名字相同；
（3）参数相同；
（4）基类函数必须有virtual 关键字。

当派生类对象调用子类中该同名函数时会自动调用子类中的覆盖版本，而不是父类中的被覆盖函数版本，这种机制就叫做覆盖。基类的对象调用基类的函数。

指向派生类的基类指针调用的也是派生类的覆盖函数。

函数隐藏

不论有无virtual关键字，当派生类函数同名，但参数不同时，基类的函数将被隐藏。

  • 非虚函数【存疑】
  • 同名
  • 并非多态性：派生类无法访问基类函数

派生类的默认成员

派生类继承了基类的实现和基类的接口，还需要增加派生类的构造/析构函数，其他新增成员。

  1. 创建时是在栈内进行的，栈有着先进后出的属性，所以先创建的后析构，后创建的先析构。

    • 创建时：先基类再派生类

      继承意味着派生类对象可以使用基类的方法，然而构造函数在完成其工作之前，对象并不存在。

      在创建派生类对象时，程序首先创建基类对象，即基类对象应当在程序进入派生类构造函数之前被创建。

    • 销毁时：先派生类再基类

    • 多继承

      调用构造函数顺序与继承列表顺序一致。

      析构时相反。

    • 如果没有缺省值，则构造函数无法成功调用基类构造函数。

      Father(int a = 0)定义缺省值；

      Son(int a):Father(0)初始化列表赋值；

      Son(int a):Father(a) + Son tmp(0)在类外给基类成员赋值构造。

  2. 拷贝构造函数

    • 使用场景

      将新的对象初始化为一个同类对象；

      按值将对象传递给函数；

      函数按值返回对象；

      编译器生成临时对象。

    • 如果在基类成员中有指针变量，必须要进行显式定义拷贝构造函数，即进行深拷贝。不然会造成同一块内存空间被析构两次的问题。

  3. 赋值操作符构造

    赋值构造不能被继承，默认采用成员成员赋值（一一对应赋值）。

    • 基类 = 派生类

      father = son; => father.operator(son);

      仅处理派生类中的基类部分，新定义部分不赋给基类。

    • 派生类 = 基类

      son = father; => son.operator(father);

      此时不能运行，除非在派生类中定义了Son(const Father&)构造函数。

赋值兼容规则

public继承条件下的特殊关系：

  1. 派生类对象可以使用基类的方法，条件是基类的方法不是私有的；
  2. 基类指针可以在不进行显示类型转换的情况下指向派生类对象；
  3. 基类引用可以再不进行显示类型转换的情况下引用派生类对象，但是基类指针或引用只能用于调用基类的方法，不能用基类指针或引用调用派生类的成员及方法。

优点：代码复用、接口多态性、灵活性、模拟现实世界

  2. 菱形继承（冗余性 二义性）

    想要的跟实际的区别：

    间接基类的成员变量在底层的派生类中会存在多份拷贝，通过底层派生类对象访问间接基类的成员变量时，会出现访问二义性。

虚继承

虚继承是面向对象编程中的一种技术，是指一个指定的基类，在继承体系结构中，将其成员数据实例共享给也从这个基类型直接或间接派生的其它类。

在内存中，虚继承后，虚基类成员在栈中的位置会存放在派生类之后。（构造函数调用顺序不变）

虚基类表不是虚函数表，虚基类表通常存储在其他位置，通过对象内存顶部的vbptr访问，虚函数表也是通过指针访问的，一般存储在对象内存的特定位置（最前面，vbtable后）。

虚基类：被虚继承的基类通常称为虚基类，虚基类只是针对虚继承，而不是针对基类本身。

虚基类指针：每个虚继承的派生类都会增加一个虚基类指针vbptr，该指针位于派生类对象的顶部。vbptr指针指向一个虚基类表vbtable。

虚基类表：存放的是各个虚基类的成员变量在子类中的内存布局中相对于派生类对象起始地址的偏移量，并且是以NULL结尾的。目的是用来解决由于从不同路径下继承而来的同一基类在子类里面会有多份拷贝而带来的问题。

当虚基类的派生类被当作基类继承时，虚基类指针vbptr也会被继承，因此底层派生类对象中成员变量的排列方式与普通继承有所不同。

在虚继承中，底层派生类的构造函数不仅负责调用直接基类的构造函数，还负责调用间接基类的构造函数。在整个对象的创建过程中，间接基类的构造函数只会调用一次。

  1. 为什么虚继承可以保证派生类中只有一个副本？

    当使用虚继承时，编译器会在内部处理多重继承的层次结构，以确保基类在派生类中的唯一性。

    • 调整内存布局：编译器会调整派生类的内存布局，以容纳虚基类子对象。这个虚基类子对象在派生类的内存中是唯一存在的。
    • 修改构造函数调用：编译器会修改派生类的构造函数，以确保虚基类的构造函数只被调用一次。无论派生类通过多少条路径继承了虚基类，其构造函数都只会沿着某一条路径被调用。其他路径上的构造函数调用会被编译器忽略或优化掉。
    • 插入虚基类指针和虚基类表：编译器会在派生类对象中插入一个虚基类指针（vbptr），该指针指向一个虚基类表（vbtable）。虚基类表包含了指向虚基类子对象的偏移量。这个机制是在运行时用于定位虚基类子对象的，但它在编译时就已经确定了虚基类子对象在派生类内存中的位置。

派生类中成员布局

详见：C++类的内存布局

  • 虚函数表存储顺序：先基类，再派生类，先声明先存储。

  • 单一继承中，基类和派生类中的虚函数个数不会影响虚函数表指针的个数。

  • 派生类的成员变量存储顺序与虚函数表中的存储顺序是一致的： 先基类再派生，先声明，先存储。

  • 大小问题

    含有虚函数的基类和派生类，计算类所占长度=成员变量长度+虚函数表指针的长度+虚基类表指针长度；

    32位下4字节，64位下8字节；

    普通函数（static以外）存储在代码区，不影响类实例的大小，通过this指针访问；

    static函数也不影响类实例的大小，但是存储在静态区，与类名相关；

    虚函数表中各个指针只占1字节。

  • 两个有虚函数的基类，VS中会有两个vfptr，g++也是。Windows下每个4字节。

    Linux中也会有两个vfptr，同时每个8字节。

  2. 私有继承和组合(has-a关系)

    私有继承是面向对象编程中继承的一种形式。在私有继承中，基类的公有成员和保护成员在派生类中成为私有成员，这意味着它们不能被派生类的子类所访问。私有继承的主要特点是它提供了基类实现细节的复用，但隐藏了基类的公有接口。派生类可以获得基类的一份备份，并获得了访问基类的公共以及保护接口的权利和重写基类虚函数的能力。在某些情况下，私有继承可以被视为组合的一种语法上的变形，例如，“汽车有一个引擎”关系可以用单一组合表示，也可以用私有继承表示。

    组合则是另一种实现代码复用的方式。在组合中，一个类（称为包含类）的对象将另一个类（称为被包含类）的对象作为自己的属性。组合的主要优点是它降低了类之间的耦合度，减少了代码冗余，并且使得代码更易于理解和维护。

    私有继承主要用于复用基类的实现细节，而组合则主要用于通过包含其他类的对象来扩展类的功能。
