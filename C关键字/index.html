C++关键字

发表于2023-06-10

static

const

inline

用于建议编译器对函数进行内联展开。内联展开是一种编译器优化技术，它会在调用函数的地方直接展开函数体，而不是通过函数调用的方式执行函数体。

  1. 减少函数调用开销：主要目的。函数调用会涉及到压栈、跳转等操作，而内联函数在编译阶段会直接将函数体插入到调用处，避免了这些开销。
  2. 提高性能： 对于短小的函数或者频繁调用的函数，避免了函数调用的开销，可以减少不必要的指令执行和内存访问。
  3. 可能导致代码膨胀： 过度使用内联函数可能会导致代码膨胀，增加可执行程序的大小。因为函数体会被复制到每个调用处，如果函数体较大或者被频繁调用，可能会导致代码量急剧增加。
  4. 建议性的： inline 关键字只是对编译器的建议，编译器不一定会采纳。编译器可能会根据实际情况决定是否对函数进行内联展开。即使不写，函数体也有可能根据优化策略展开。

内联函数的定义通常放在头文件中，这样可以保证在每个使用了该函数的地方都能够进行内联展开。

volatile

  • 随时改变：此变量最好不要连续取两次

  • 禁止编译器对变量进行优化

    所修饰的变量可能会在当前程序的外部被改变，阻止编译器因误认为某段代码无法被代码本身多改变，而造成的过度优化。每次读取变量时，不要使用缓存中的值，从内存处重新读取。

  • 顺序执行：

    编译器和处理器在处理变量时会按照程序中的顺序执行相关操作，而不会重排序优化。

    在多线程编程中，确保变量的内存可见性，一个线程修改了变量后，另一个线程可立即看到。

    但不能保证线程安全，仍然需要加锁。

explicit

用于修饰类的构造函数。使用 explicit 关键字可以防止编译器使用该构造函数进行隐式类型转换。

不能使用int类型值来初始化MyClass类对象。

它明确了对象的构造方式，避免了一些可能导致不必要错误的隐式类型转换，提高代码的安全性和可读性。

不使用时可能的错误：

  • 意外的隐式类型转换

    在 main 函数中，试图使用 Distance d2 = 2000; 这样的语句进行隐式类型转换，编译器会将 int 类型的 2000 隐式地转换为 Distance 类型的对象，从而调用了 Distance(int) 构造函数。

  • 歧义的函数调用

    编译器将整数 2000 隐式地转换为 Distance 对象，然后错误地匹配到了接受 int 参数的 printDistance(int) 函数，而不是原本预期的调用 printDistance(const Distance&) 函数。

  • 安全性问题

  • 代码可读性问题

mutable

修饰类的成员变量：使用 mutable 关键字修饰的成员变量可以在常量成员函数中被修改，即使这些函数被声明为 const。

作用：允许对象在常量上下文中修改其状态。

const成员函数可以修改mutable成员变量。

exteren

用于声明一个变量或函数的外部链接性（external linkage）。extern 声明告诉编译器，被声明的变量或函数是在其他文件中定义的，其定义在当前文件之外。

  1. 声明外部变量
  2. 声明外部函数
  • 分离声明和定义
  • 起链接器作用
  • 避免多重定义
  • 避免重复编译，提高效率

enum

枚举类型是一种用户自定义的数据类型，用于定义一组具名的整型常量。使用枚举类型可以提高代码的可读性和可维护性，使得程序中的常量具有更好的语义性。

虚函数相关

详见虚函数内容。

virtual/override/final

  1. virtual： virtual 关键字用于声明虚函数，即可以在派生类中被重写的基类函数。它使得在运行时能够动态地确定调用的是哪个版本的函数。
  2. override： override 关键字用于在派生类中显式地声明对基类虚函数的重写。使用 override 可以提高代码的可读性，并在派生类中确保正确地重写了基类的虚函数。
  3. final： final 关键字用于在基类中显式地声明某个虚函数不可被派生类重写。在派生类中尝试重写被声明为 final 的虚函数会导致编译错误。

typeid

typeid 运算符用于获取一个表达式的运行时类型信息。在面向对象的设计中，常用于判断对象的实际类型，与虚函数一起使用可以实现动态多态。
