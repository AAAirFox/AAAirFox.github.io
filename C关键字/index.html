C++关键字

发表于2023-06-10

static

https://blog.csdn.net/weixin_45031801/article/details/134215425

作用主要是控制变量的存储方式和作用范围。

  1. c

    修饰全局变量：同一文件内可见（全局变量原本可以在其他文件extern，指定为内部链接后，无法解析）

    修饰局部变量：改变存储位置（栈→静态区），值不因函数终止而丢失（函数首次调用时初始化）

    修饰函数：同一文件内可见

  2. c++

    修饰类的成员变量：该类所有对象的此成员只有一个实例，属于整个类，所以不依赖于任何实例的创建，不能依赖于任何对象的构造函数【不能在类中初始化，除非添加了const：因为值在编译时是已知的，初始化后不会改变】

    修饰类的成员函数：只能访问static修饰的成员，以及全局变量

static变量存储在静态存储区，在main前初始化，如果没有显示定义则默认为0（全局变量的属性）.

static函数存储在代码区，可互相调用，其他函数可以访问static函数，但s函数不能访问其他函数，不能在类中声明+类外重复static。

优点：不被其他文件所用，可定义相同名字在其他文件中，不会冲突。类中则是可实现共享。

  • this指针问题

    this指针是一个隐含的指针，它指向调用成员函数的对象实例。当成员函数被调用时，编译器会自动将对象的地址作为this指针传递给该函数。

    静态成员函数不存在this指针，调用通过class::func，无法访问非static成员;

  • 不能被声明为const，通过this指针调用：

    在C++中，this指针本身不能被声明为const，因为this指针的本质是指向当前对象实例的指针，它必须能够被修改以支持对象的成员函数通过该指针访问和修改对象的成员。将this指针声明为const会限制它的灵活性，使得成员函数无法正确地操作对象的状态。

    此外，尝试通过const的this指针来调用成员函数也是没有意义的，因为这会与const的语义相冲突。如果一个成员函数通过const的this指针被调用，那么它将无法修改对象的任何成员变量（无论是静态还是非静态），因为const指针指向的对象本身被认为是不可变的。然而，这样的限制与C++中成员函数通常能够修改对象状态的能力相违背。

    在C++中，我们通常通过将成员函数声明为const来表明该函数不会修改调用它的对象的状态。当成员函数被声明为const时，编译器会隐式地将this指针视为指向const对象的指针，从而确保该函数不会修改任何非静态成员变量（除非它们被声明为mutable）。但是，这并不意味着this指针本身是const的；它仍然指向一个对象，只是该对象在成员函数内部被视为不可变的。

  • 静态成员函数可以被继承，但调用时仍然调用基类函数：静态成员函数没有虚函数机制来支持运行时动态绑定。

    静态成员和类的普通成员一样，也有public、protected、private3种访问级别，也可以具有返回值。

  • 静态成员函数没有虚函数机制来支持运行时动态绑定。

    虚函数机制依赖于每个对象实例中的虚函数表（vtable）和虚指针（vptr），它们用于在运行时确定应该调用哪个函数。然而，静态成员函数不属于任何特定的对象实例，因此它们没有与之关联的vptr和vtable。因此，静态成员函数不能被声明为虚函数，也不能通过虚函数机制实现动态绑定。

  • 静态成员函数不能声明为虚函数，不能实现动态绑定：不与实例关联。

  • 构造函数、析构函数不能为静态函数：

    构造函数必须能够访问对象的成员变量，并且通常通过this指针来操作这些成员。静态成员函数则不与任何特定的对象实例关联，它们没有this指针，因此无法访问非静态成员变量。因此，构造函数不能是静态的。

    析构函数也与特定的对象实例相关，并且需要能够访问对象的成员变量来执行清理工作。因此，析构函数也不能是静态的。

const

作用主要是禁止修改。

  • 变量：值不变，声明时初始化
  • 指针：指针常量const *指向的值不变，常量指针* const
  • 函数形参：参数不可变
  • 函数返回值：限定为右值
  • 成员函数：不能改变对象的变量，不能调用非const成员，只能被const成员调用，不加static
  • 成员变量：不能被改变，只能在初始化列表初始化。（不能在class中：const成员变量的值在对象生命周期中保持不变，因此必须在对象构造时立即设置，否则违反不变性原则）

  1. 不能声明为const的：

    static函数：没有this指针

    构造函数：const只能访问const成员，构造函数需要初始化成员。

  2. 全局const变量放在常量区，局部const变量放在战区，未被使用的放在符号表（仅编译时的）

  3. 与define的区别：
    数据类型、是否可调试、处理阶段、是否分配内存

  4. const对象不可以调用非const成员函数：防止权限放大

  5. 声明成const的虚函数成员有什么作用?

    保持对象状态的不变性：

    不会修改调用它的对象的任何非静态成员变量（除非这些变量被声明为mutable）。对保持对象状态不变的操作非常有用，比如获取器（getter）函数。这样的设计有助于维护对象的封装性和状态的不变性，确保对象的状态不会在不需要的时候被意外修改。

    正确的多态行为：

    多态性允许通过基类指针或引用来调用派生类中的重写函数。当基类中的虚函数被声明为const时，任何派生类中重写这个函数的实现也必须是const的。这确保了通过基类指针或引用调用这个虚函数时，无论实际对象是基类还是派生类的实例，都不会修改对象的状态。这有助于保持多态行为的正确性和一致性。

  6. const和static

constexpr

指定一个变量或函数是“常量表达式”。主要目的是允许在编译时计算值，而不是在运行时。

右侧一定是一个常量表达式。

即使一个函数是constexpr的，编译器也不一定会在编译时评估它。这取决于函数的使用上下文，比如当main中使用someFunction函数，可能运行时调用。

  • 变量

  • 函数

    能⽤于常量表达式的函数。

    1. 它必须有且只有一个返回语句。
    2. 它返回的类型也必须是常量表达式。
    3. 它不能有任何循环、递归或其他可能在运行时影响结果的行为。
    4. 它的所有参数也必须是常量表达式（如果有的话）。
    5. 函数中不能调用非constexpr函数。

    为了可以在编译过程展开，constexpr函数被隐式转换成了内联函数。但并不总是内联的，要强制内联，可以使用inline说明符。

    constexpr和内联函数可以在程序中多次定义，⼀般定义在头⽂件。

  • 构造函数

    构造函数不能说const，但字⾯值常量类的构造函数可以是constexpr。

    constexpr构造函数必须有⼀个空的函数体，即所有成员变量的初始化都放到初始化列表中。对象调⽤的成员函数必须使⽤ constexpr 修饰。

  1. 优点

    • 性能：由于constexpr的值在编译时就已经确定，因此没有运行时开销。
    • 类型安全：编译器可以在编译时检查constexpr表达式，这有助于捕获类型错误。
    • 优化：编译器可以对constexpr表达式进行优化，因为它们是常量。

  2. constexpr和const的区别

    const 表⽰“只读”的语义，constexpr 表⽰“常量”的语义

    constexpr 只能定义编译期常量，⽽ const 可以定义编译期常量，也可以定义运⾏期常量。

    你将⼀个成员函数标记为constexpr，则顺带也将它标记为了const。如果你将⼀个变量标记为constexpr，则同样它是const的。但相反并不成⽴，⼀个const的变量或函数，并不是constexpr的。

define

简单替换、宏定义

#号：替换成string

##号：连接两个变量

  1. define和const区别

    数据类型、是否可调试、处理阶段、是否分配内存

  2. define和typedef区别

    • define没有类型检查，typedef需要判断
    • define预处理阶段，typedef编译/运行阶段
    • define没有内存，typedef在静态存储区中分配空间，运行过程中内存中只有一个拷贝
    • type无法防止头文件重复引用

  3. define和inline区别

    • define只是简单替换，inline将内联函数编译完成生成了函数体直接插入，减少了压栈过程/跳转/返回的操作，没有额外开销。
    • define没有类型检查，inline进行类型检查
    • define一定被处理，inline只是一种请求

typedef

类型定义：将一种数据类型定义为某一个标识符，在程序中使用标识符来实现相应的数据类型。

如果用它定义指针typedef int* int_ptr，那么替换时不会像define一样INT_PTR a,b;时b出现错误。

用法：

  • 基本数据类型

    typedef unsigned char uchar

  • 数组类型

    typedef int array [2] => array a == int a[2]

    typedef int array [M][N] => array a == int a[M][N]

  • 指针类型

    typedef int *ptr [M] => ptr p == int *p[M]

  • 定义结构体别名

  • 定义类类型

  • 函数指针

inline

用于建议编译器对函数进行内联展开。内联展开是一种编译器优化技术，它会在调用函数的地方直接展开函数体，而不是通过函数调用的方式执行函数体。

  1. 减少函数调用开销：主要目的。函数调用会涉及到压栈、跳转等操作，而内联函数在编译阶段会直接将函数体插入到调用处，避免了这些开销。
  2. 提高性能： 对于短小的函数或者频繁调用的函数，避免了函数调用的开销，可以减少不必要的指令执行和内存访问。
  3. 可能导致代码膨胀： 过度使用内联函数可能会导致代码膨胀，增加可执行程序的大小。因为函数体会被复制到每个调用处，如果函数体较大或者被频繁调用，可能会导致代码量急剧增加。
  4. 建议性的： inline 关键字只是对编译器的建议，编译器不一定会采纳。编译器可能会根据实际情况决定是否对函数进行内联展开。即使不写，函数体也有可能根据优化策略展开。

内联函数的定义通常放在头文件中，这样可以保证在每个使用了该函数的地方都能够进行内联展开。

C++中的编译限制：

  • 不能存在任何形式的循环语句
  • 不能存在过多的条件判断语句
  • 函数体不能庞大
  • 声明必须在调用之前

volatile

  • 随时改变：此变量最好不要连续取两次

  • 禁止编译器对变量进行优化

    所修饰的变量可能会在当前程序的外部被改变，阻止编译器因误认为某段代码无法被代码本身多改变，而造成的过度优化。每次读取变量时，不要使用缓存中的值，从内存处重新读取。

  • 顺序执行：

    编译器和处理器在处理变量时会按照程序中的顺序执行相关操作，而不会重排序优化。

    在多线程编程中，确保变量的内存可见性，一个线程修改了变量后，另一个线程可立即看到。

    但不能保证线程安全，仍然需要加锁。

explicit

用于修饰类的构造函数。使用 explicit 关键字可以防止编译器使用该构造函数进行隐式类型转换。

不能使用int类型值来初始化MyClass类对象。

它明确了对象的构造方式，避免了一些可能导致不必要错误的隐式类型转换，提高代码的安全性和可读性。

不使用时可能的错误：

  • 意外的隐式类型转换

    在 main 函数中，试图使用 Distance d2 = 2000; 这样的语句进行隐式类型转换，编译器会将 int 类型的 2000 隐式地转换为 Distance 类型的对象，从而调用了 Distance(int) 构造函数。

  • 歧义的函数调用

    编译器将整数 2000 隐式地转换为 Distance 对象，然后错误地匹配到了接受 int 参数的 printDistance(int) 函数，而不是原本预期的调用 printDistance(const Distance&) 函数。

  • 安全性问题

  • 代码可读性问题

mutable

修饰类的成员变量：使用 mutable 关键字修饰的成员变量可以在常量成员函数中被修改，即使这些函数被声明为 const。

作用：允许对象在常量上下文中修改其状态。

const成员函数可以修改mutable成员变量。

enum

枚举类型是一种用户自定义的数据类型，用于定义一组具名的整型常量。使用枚举类型可以提高代码的可读性和可维护性，使得程序中的常量具有更好的语义性。

虚函数相关

详见虚函数内容。

virtual/override/final

  1. virtual： virtual 关键字用于声明虚函数，即可以在派生类中被重写的基类函数。它使得在运行时能够动态地确定调用的是哪个版本的函数。
  2. override： override 关键字用于在派生类中显式地声明对基类虚函数的重写。使用 override 可以提高代码的可读性，并在派生类中确保正确地重写了基类的虚函数。
  3. final： final 关键字用于在基类中显式地声明某个虚函数不可被派生类重写。在派生类中尝试重写被声明为 final 的虚函数会导致编译错误。

typeid

typeid 运算符用于获取一个表达式的运行时类型信息。在面向对象的设计中，常用于判断对象的实际类型，与虚函数一起使用可以实现动态多态。

struct

  1. C和C++中的区别？

    • 成员函数的支持：
      在 C++ 中，struct 可以包含成员函数（包括构造函数和析构函数）。这使得 C++ 的 struct 更像是一个类，尽管其默认访问权限是公共的（public），而类的默认访问权限是私有的（private）。
      相比之下，C 中的 struct 只能包含数据成员，不能包含函数。

    • 访问控制：
      在 C++ 中，你可以为 struct 的成员指定访问控制（public、private、protected）。
      在 C 中，struct 的所有成员默认都是公共的（public），并且没有提供其他访问控制选项。

    • 继承：
      C++ 中的 struct 支持继承，这意味着一个 struct 可以从另一个 struct 或类继承其成员。
      C 中的 struct 不支持继承。

    • 初始化列表：
      在 C++ 中，struct 的构造函数可以使用初始化列表来初始化其成员。
      C 没有构造函数或初始化列表的概念。

    • 模板：
      C++ 支持模板 struct，这使得你可以创建泛型数据结构，如模板类一样。
      C 不支持模板。

    • 操作符重载：
      在 C++ 中，你可以为 struct 重载操作符，从而定义自定义的操作符行为。
      C 不支持操作符重载。

    • 类型别名：
      在 C++ 中，你可以使用 typedef 或 using 关键字为 struct 创建类型别名。
      C 也支持 typedef，但用法可能略有不同。

new/malloc/delete/free

malloc内存分配

malloc申请不是真正的在物理内存上申请内存，它也是在虚拟内存的中申请。实际上malloc()函数 并不是一个系统调用，而是 C 库里的函数（不是操作符），它用于动态分配内存。当我们使用malloc 申请内存的时候，malloc底层会有两种方式向操作系统申请堆内存。malloc其原型void *malloc(unsigned int num_bytes);.

方式一：这种方式就是通过调用brk()函数，将堆顶的指针想高地址移动所需大小个。

方式二：通过 mmap() 系统调用中「私有匿名映射」的方式，在文件映射区分配一块内存，也就是从文件映射区“偷”了一块内存。

什么时候使用brk什么时候使用mmap呢？

  • 如果用户分配的内存小于 128 KB，则通过 brk() 申请内存；
  • 如果用户分配的内存大于 128 KB，则通过 mmap() 申请内存；

注意：不同的 glibc （c运行库）版本可能阈值不同。

  1. malloc申请空间一定就是我们指定的空间大小吗？

    malloc并不是只申请指定空间的大小，它会申请比指定空间大一点的空间。

    • 额外的空间用来记录管理信息分配块长度，指向下一个分配块的指针等等。如果程序不小心写入了一个已分配内存块的尾端之外，即发生了缓冲区溢出，它很可能会覆盖掉后面内存块的管理信息。

      调用malloc函数，它会返回一个指针，这个指针指向申请空间的起始位置。另外会多申请一块空间head，来保存我们申请这片空间的信息，比如我们申请空间的长度是多少，起始地址。 多申请这片空间的目的是为了方便free的使用。

      不同操作系统中”head“的数据结构也是不一样的，一般是一个联合体，内部嵌套一个结构体，结构体中包含两个成员，一个是指向下一个头部的指针，一个表示堆内存的大小。

    • 同时内存管理机制会预分配更大的空间作为内存池。

  2. 为什么不只使用mmap来申请空间？

    频繁通过 mmap 分配的内存，不仅每次都会发生运行态的切换，还会发生缺页中断（在第一次访问虚拟地址后），这样会导致 CPU 消耗较大。

    • 向操作系统申请内存，是要通过系统调用的，执行系统调用是要进入内核态的，然后在回到用户态，而用户态到内核态的切换是会耗费不少时间的，所以，申请内存的操作应该避免频繁的系统调用，如果都用 mmap 来分配内存，等于每次都要执行系统调用。（mmap就是一个系统调用嘛）。
    • 每次释放的时候，都会归还给操作系统，于是每次 mmap 分配的虚拟地址都是缺页状态的，然后在第一次访问该虚拟地址的时候，就会触发缺页中断。因为归还给操作系统了虚拟内存和物理内存的映射关系都不在了，而brk申请然后释放是将这块内存块重新标记为可用，映射关系还在。

    brk可以改进。

  3. 为什么不只使用brk来申请空间？

    brk()申请的空间，是将堆顶指针往高地址移动，而释放的时候它不会归还给操作系统，而是将该内存块标记为可用，然后放入内存池。（如果50kb，超过内存池中大小了，就再申请一个）

    所有使用brk()申请空间，特别是在数据大小不一零碎的时候，特别容易造成堆内存产生越来越多的不可用的小碎片，而导致“内存泄露”。而这种“泄露”现象使用 valgrind 是无法检测出来的。（虽然内存管理器可能会重组小内存，但是也是需要消耗一定资源的）

free释放空间

malloc申请空间时的“head”，里面保存了我们申请这片空间的大小，所以free传入需要释放空间的首地址，通过首地址向前偏移操作系统就能读到”head”保存的信息，知道释放空间是多大了。

free释放内存后是否会归回给操作系统，也是取决于malloc申请时具体调用的是哪一种方法申请的。（brk放入内存池，mmap还给操作系统）

如果内存还在，你释放了去打印它的地址它会出现一个具体的地址，比如2F530010。所以释放后一般要把ptr = nullptr。如果内存真正的释放了，你去打印是不会出现比如2F530010，这样的地址的。

malloc/realloc/calloc

  • malloc函数其原型：

    void *malloc(unsigned int num_bytes)。num_byte为要申请的空间大小，需要我们手动的去计算，如int *p = (int *)malloc(20*sizeof(int))。

    申请后空间的值随机，不会初始化。

  • calloc函数其原型：

    void *calloc(size_t n, size_t size)。其比malloc函数多一个参数，并不需要人为的计算空间的大小，比如要申请20个int类型空间，会int *p = (int *)calloc(20, sizeof(int)），不需要人为计算。

    calloc在申请后，对空间逐一进行初始化，并设置值为0。

  • realloc函数和上面两个有本质的区别，其原型void realloc(void *ptr, size_t new_Size)。
    用于对动态内存进行扩容(及已申请的动态空间不够使用，需要进行空间扩容操作)，ptr为指向原来空间基址的指针， new_size为接下来需要扩充容量的大小。

总结：

​ malloc：一个参数，需要手动计算空间大小，不会初始化，空间的值随机。

​ calloc：两个参数，不需要手动计算，会初始化为0。

​ realloc：是用来扩容的。

  1. 为什么不直接使用calloc?

    不需要我们手动计算申请空间的大小，但计算机需要计算；calloc还会一一初始化为0，这导致了它的效率没有malloc快，在某些不需要初始化的场合没有malloc好。

  2. realloc是如何扩容的？

    申请空间时，内存空间管理器会预申请比需要空间大的一块区域，以便后面更快的申请。

    所以当申请空间size：

    • 比预留空间小：（即比原空间后面的预申请空间小）直接在预申请的空间中开辟一块，返回原空间基地址。
    • 比预留空间大：系统将重新申请一片空间，将原空间数据拷贝过去，返回新空间地址。
    • 申请size非常大：申请失败，realloc返回空指针，原来空间不会被释放。
    • 扩容后的内存空间较原空间小：将会出现数据丢失，如果直接realloc(p, 0);相当于free(p)。

new和delete

new和malloc没有本质上的区别，都是在堆区申请内存，只不过new会调用构造函数，delete会调用析构函数。而且new和delete不是C库函数，而是一个操作符。

  1. 底层实现

  2. 过程

    new:

    • 调用operator new[]函数，实际在operator new[]中调用operator new函数完成N个对象空间的申请。
    • 调用N次构造函数

    delete:

    • 调用N次析构函数，完成N个对象中资源的清理
    • 调用operator delete[]释放空间，实际在operator delete[]中调用operator delete来释放N个对象空间

  3. new[]申请的空间可用使用delete释放吗？

    如果正常使用delete[]释放空间，它会根据“head”的信息，知道需要释放的自定义数据类型的个数为5，然后调用5次operator delete和5次析构函数来释放它。

    delete和delete[]释放内存的过程区别就是delete只调用一次，delete[]调用多次析构函数。

    • 自定义类型：（因为类中也可能使用new去申请空间），如果使用delete去释放，仅仅只是数组第一个元素调用了析构函数，其他元素没有调用析构函数。所有只有第一个元素正确得到了释放，如果其他元素对象的内部也使用了new，那就会有很严重的后果C++并没有定义这种情况的处理机制。
    • 内置数据类型：因为内置数据类型释放不需要调用析构函数，所以使用delete也是可以释放成功的。

区别总结

new分配内存的位置在自由存储区，自由存储区（可堆可栈区） 是C++中动态分配和释放对象的一个概念，通过new分配的内存区域可以称为自由存储区，通过delete释放归还内存。自由存储区可以是堆、栈等，具体是在哪个区，主要还是要看new的实现以及C++编译器默认new申请的内存是在哪里。但是基本上，很多C++编译器默认使用堆来实现自由存储，运算符new和delete内部默认是使用malloc和free的方式来被实现，说它在堆上也对，说它在自由存储区上也正确。因为在C++中new和delete符号是可以重载的，我们可以重新实现new的实现代码，可以让其分配的内存位置在栈区等。而malloc和free是C里的库函数，无法对其进行重载。

new操作符内存分配成功时，返回的是对象类型的指针，类型严格与对象匹配，无须进行类型转换，故new是符合类型安全性的操作符。而malloc内存分配成功则是返回void * ，需要通过强制类型转换将void*指针转换成我们需要的类型。所以在C++程序中使用new会比malloc安全可靠。
